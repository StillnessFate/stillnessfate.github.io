Development Participation Guide


Anyone can contribute by participating in the development of a project!

NewSTyle is an open source project open to everyone.

But please observe some precautions.


Issues

We hope you have a lot of communication through the issues tab in Github.

Create a new issue if you have any bugs or questions you've found

But be sure to choose the right label to avoid confusion.


Pull requests

Before making a pull request, make sure that the state of the project is the latest version and check that there are no conflicts.

And if you belong to Collaborators, do not pull directly into the master branch.


License

If you add a module with a different license, make sure that it does not conflict with the current license, and please notify the administrator before using them.

This is very important for open source projects.


Code style

Do not use the <style> tag inside an HTML document. Stylesheets must be written in external CSS. Make HTML documents as concise and easy to view as possible.

It is also recommended that you do not write directly inside the HTML with JavaScript.

Import the Stylesheet file from the head, and import the Javascript file from the end of the body.

We recommend that you follow the following code style guidelines by default:

https://google.github.io/styleguide/htmlcssguide.html https://google.github.io/styleguide/jsguide.html

Resource

Limit the size of your resources (images, etc.) to a maximum of 1 MB to avoid slowing the page's loading.
